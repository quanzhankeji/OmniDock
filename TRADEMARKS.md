## Uses That Do Not Require Separate Permission

You may:

- Refer to the OmniDock project by name in factual articles, reviews, issue reports, and technical discussions.
- Link to the official repository or an official distribution page.
- State truthfully that another project is based on, compatible with, or derived from OmniDock, provided the wording does not imply endorsement.
- Use the name to identify an unmodified copy obtained from an official company distribution channel.

## Uses That Require Written Permission

Written permission is required to:

- Use OmniDock or a confusingly similar term as the name of a modified product, company, service, domain, or application listing.
- Use the official logo or app icon for a fork, redistributed build, or unrelated product.
- Suggest sponsorship, certification, partnership, or endorsement by Chengdu Quanzhan Technology Co., Ltd.
- Present a modified or independently signed build as an official OmniDock release.

## Forks and Modified Builds

A publicly distributed fork or modified build should:

- Use a distinct product name, bundle identifier, logo, and app icon.
- Clearly state that it is an independent project derived from OmniDock.
- Avoid visual presentation that could reasonably be confused with an official release.
- Preserve copyright and GPL notices required by the source license.

These branding requirements do not limit the rights to use, modify, or distribute source code granted by GPL v3.
